Building a Searchable André Hueston Mack Wine Review Database

This project turned public wine-review material into a structured static research product: bottle cards with source links, captured notes, rank context, buy-search links, and filterable fields.

The Research Question

We needed a defensible way to say how long André Hueston Mack has been evaluating and recommending wine, then connect that credibility claim to a searchable database of his public wine discussion. The public-record framing matters because the site is not claiming access to private tasting notes or unpublished ratings.

The copy used on the published site is intentionally conservative: “Sommelier and winemaker André Hueston Mack has been professionally evaluating and recommending wines for more than 23 years. We spent 3.5 hours reviewing his reviews.”

Method

1. Source Inventory

Public videos, article references, podcast pages, television references, and competition materials were inventoried first. Each source was classified by whether it produced bottle-level data, general context, or no clean additional bottle list.

2. Bottle Extraction

Named wines, wine styles, pairings, judgments, and explicit discussions were converted into normalized cards. Each card preserves source context instead of flattening every mention into the same kind of rating.

3. Interface Structure

The static page organizes cards with practical controls: dollar rank, beanie rank, thumbs verdict, wine type, and review source. Buy-search and review-search links keep the page useful without pretending to be a retail catalog.

4. Verification Layer

Outside notes and rating context are shown only where a public outside source was captured. Missing outside notes are not hidden in the data model, but the public hero no longer emphasizes the backlog as a front-page stat.

Implementation Notes

  • The source record lives in Markdown and structured JSON support files.
  • A Node.js generator builds the wine-card site and the build-notes page from the same data model.
  • The generated HTML is static, so it can be published to Labs without a server or database.
  • The build notes preserve the version log, method decisions, and live elapsed timer for project transparency.
