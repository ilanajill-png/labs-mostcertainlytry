# Building the Reviewer Panel for the R1 Video Project

The R1 video project used a structured reviewer panel to turn a short personal-agent promo into a stronger, more publishable campaign asset. Instead of relying on one broad critique, the project created role-based reviewers with distinct jobs: awards craft, platform performance, business credibility, investor signal, partner usability, creator appeal, plain-language comprehension, accessibility, and audio identity.

The reviewer system became part of the production workflow itself. Each round exposed a different weakness: vague payoff, missing human proof, synthetic voice quality, generic audio identity, cropped screenshots, mobile safe-zone problems, and small workflow labels.

## Core Pattern

1. Build a candidate cut.
2. Create or update a links page for the version.
3. Ask the reviewer panel to evaluate the cut from assigned lenses.
4. Summarize consensus and disagreements.
5. Convert critique into concrete production changes.
6. Re-render the next candidate.
7. Run QA against platform exports and safe zones.

## Final Outcome

- Average final score: 9.1/10.
- Reviewer preference: 11/11 preferred V19 over previous versions.
- Export QA: 15/15 platform exports passed duration, dimensions, and audio checks.
- Blocking concerns: none.
